Chapter II

Methodology

Methodology is the systematic approach, techniques, and processes used to carry out the study and develop the system. This chapter follows the institutional outline (Parts A–E) and aligns each section with the live Project OS modules and architecture diagrams.

Software Process Model

The project adopts an Agile, iterative software development methodology. Agile is justified by the exploratory nature of the AI behavioral-monitoring component, the need for continuous integration between the React frontend, Django backend, and the computer-vision pipeline, and the capstone timeline that benefits from incremental, demonstrable milestones. Work is organized into sprints mapped to the Work Breakdown Structure and visualized in the Gantt chart.

Reference models considered: Agile, Waterfall, Scrum, Kanban, Spiral, V-Model, Iterative, Incremental, RAD, DevOps, Lean, Extreme Programming (XP), Prototype, and Big Bang.
