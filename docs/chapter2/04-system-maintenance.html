System Maintenance

Post-deployment security and maintenance plans for sustaining Knowing Eye in academic production environments.

Systems Security Plan

Security controls are implemented across authentication, transport, application, and data layers. The plan below defines preventive, detective, and corrective measures.

1. Authentication & Authorization

ControlImplementationMaintenance Action
Password storageDjango PBKDF2 hashingEnforce minimum length; rotate on breach
Session tokensJWT access (60 min) + refresh (7 days) with rotation & blacklistMonitor token TTL env vars; revoke on compromise
RBACADMIN / EXAMINEE roles on all API and WS routesAudit new endpoints for permission classes
WebSocket authJWT in query string validated by JWTAuthMiddlewareUse WSS only in production

2. Transport & Network Security

  • Production: TLS 1.2+ via Nginx; SECURE_SSL_REDIRECT=True, HSTS enabled when DEBUG=False.
  • CORS restricted to known frontend origins in production (CORS_ALLOWED_ORIGINS).
  • Rate limiting: DRF throttling (60/min anon, 1200/min authenticated).
  • ALLOWED_HOSTS and AllowedHostsOriginValidator prevent host-header attacks.

3. Application & Data Security

AssetThreatMitigation
Exam contentUnauthorized accessJWT + exam ownership checks; draft exams hidden from examinees
Behavior logsData leakageSession-scoped queries; examinees see own data only
Face embeddingsBiometric misuseStored per session only; not shared across users; deletable with session
Media uploadsMalicious filesAvatar upload via DRF parser; size/type validation recommended
SecretsExposure in repo.env gitignored; DJANGO_SECRET_KEY required in prod

4. Security Monitoring & Incident Response

  1. Review backend/logs/knowing_eye.log for 401/403 spikes and 500 errors.
  2. Monitor unresolved behavior alerts via admin dashboard.
  3. On suspected breach: rotate DJANGO_SECRET_KEY, invalidate JWT blacklist, force password reset for affected accounts.
  4. Apply Django security patches monthly; dependabot or manual pip audit / npm audit.

Operational Procedures

Database Backup

  • PostgreSQL: daily pg_dump to secure storage
  • SQLite (dev): copy backend/db.sqlite3 before migrations
  • Retention: 30 days rolling; verify restore quarterly

Application Deployment

  • python manage.py migrate before each release
  • python manage.py collectstatic for admin assets
  • Restart Daphne workers after backend deploy
  • Rebuild frontend: npm run build → serve dist/

AI Pipeline Maintenance

  • Update backend/ai/config/pipeline.yaml thresholds per proctoring policy
  • Refresh MediaPipe models in backend/ai/models/ if URLs change
  • Re-test with python manage.py test ai.tests after CV dependency updates

Log & Storage Hygiene

  • Rotating log: knowing_eye.log (5 MB × 5 backups)
  • Archive old exam sessions per institutional retention policy
  • Clear stale in_progress sessions manually or via admin terminate
