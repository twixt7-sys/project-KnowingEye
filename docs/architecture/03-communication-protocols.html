REST & WebSocket Communication

2.1.3.1 REST APIs for CRUD Operations

All REST endpoints return JSON. Authenticated requests include Authorization: Bearer <access_token>. Pagination uses DRF standard page/size query parameters where applicable.

Authentication Module — /api/auth

MethodPathAuthDescription
POST/token/NoLogin → access + refresh JWT
POST/token/refresh/NoRefresh access token
POST/register/NoCreate user account
GET/profile/me/YesCurrent user profile
PATCH/profile/update_profile/YesUpdate profile fields
POST/profile/avatar/YesMultipart avatar upload
POST/profile/change-password/YesChange password

Exams — /api/exams

MethodPathRoleDescription
GET/POST/Auth / ADMINList or create exams
GET/PUT/PATCH/DELETE/{id}/Auth / ADMINExam CRUD
POST/{id}/publish/ADMINPublish exam
POST/{id}/archive/ADMINArchive exam
GET/POST/{id}/questions/Auth / ADMINQuestion management

Sessions — /api/sessions

MethodPathDescription
POST/start/Start exam session (returns UUID)
POST/{uuid}/submit/Submit answers, calculate score
POST/{uuid}/terminate/Admin force-terminate
GET/{uuid}/logs/Session audit trail

Monitoring (REST fallback) — /api/monitoring

MethodPathDescription
GET/health/Public health check + pipeline_mode
POST/frame/Analyze single base64 JPEG frame
POST/enroll/Enroll reference face for session

2.1.3.2 WebSocket for Real-Time Monitoring

Implemented via Django Channels AsyncJsonWebsocketConsumer. JWT passed as query parameter: ?token=<access>.

Endpoints

URLConsumerAccess
/ws/monitoring/{session_uuid}/MonitoringConsumerSession owner (examinee) or ADMIN
/ws/monitoring/alerts/AdminAlertsConsumerADMIN only — global alert fan-in
