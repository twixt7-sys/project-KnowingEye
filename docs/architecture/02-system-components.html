System Components — Detailed Specification

Expanded definitions of the four primary components plus internal backend and frontend structure (§2.2–2.3).

2.1.2.1 Frontend Component

React 18ViteTypeScriptTailwindshadcn/uiRecharts

Purpose

Delivers all user-facing functionality: authentication, role-based dashboards, exam creation and taking, live monitoring views, behavioral alert display, reports, and profile management.

Architecture

Feature-Based Modular Architecture with a thin pages/ routing layer, features/ domain modules, core/ providers and config, and shared/ reusable UI.

Core Providers

  • AuthProvider — JWT storage, login/logout, token refresh
  • ThemeProvider — Dark/light mode
  • QueryProvider — TanStack React Query for server state
  • WebSocket integration — via use-monitoring hook per session

2.1.2.2 Backend Component

Django 6DRFSimpleJWTChannelsDaphne ASGI

Purpose

Central application server: REST API, WebSocket consumers, business logic, authorization, and coordination with the AI adapter and database.

Feature Modules

ModuleModelsAPI PrefixResponsibilities
authenticationUser/api/auth/JWT, register, profile, avatar, RBAC

sessionExamSession, Response, SessionLog/api/sessions/Start, submit, terminate, scoring

behaviorBehaviorLog, Alert/api/behavior/Persist AI output, alert resolution
reports— (aggregations)/api/reports/Summary, timeseries, CSV export

Service–Repository Pattern

Each feature encapsulates: models (ORM entities) → services (business rules) → views/consumers (HTTP/WS entry) → serializers (DTO mapping). Example: persist_analysis() in behavior services writes logs/alerts from AI output.
