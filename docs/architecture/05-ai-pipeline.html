Chapter 5 · §2.1.2.3 AI Module

Computer Vision & Behavior Analysis Pipeline

Module Overview

The AI module lives in backend/ai/knowing_eye/ and is integrated into Django through backend/ai/adapter.py. It processes webcam frames and returns structured metrics, behavioral events, and alerts without storing raw video in the database.

Pipeline ModeCondition
productionML dependencies installed (mediapipe, ultralytics, etc.)
stubDeterministic fallback when imports fail
disabledKE_ENABLE_PIPELINE=False in settings

Analysis Output Schema

Metrics (0–100%)

face_pct, gaze_pct, posture_pct, identity_pct, object_pct, overall_compliance_pct

Events

Array of { event_type, score, confidence, metadata }

Alerts

Array of { type, severity (low|medium|high), message, metric_pct }

Integration Flow

  1. WebSocket consumer receives base64 frame
  2. decode_base64_image() → BGR numpy array
  3. analyze_frame_bgr(frame, session_id) → pipeline
  4. persist_analysis() writes BehaviorLog and Alert rows
  5. JSON analysis + alerts returned to client; alerts broadcast to admin group
