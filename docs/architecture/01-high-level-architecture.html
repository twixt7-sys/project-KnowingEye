Chapter 1 · §2.1 High-Level Architecture Design

High-Level System Architecture

Defines the overall architectural pattern, primary system components, inter-component communication, and deployment model for the Knowing Eye platform.

2.1.1 Define System Architecture

The Knowing Eye platform adopts a Feature-Based Modular Monolith with an internal Layered Architecture. This decision balances capstone project scope, team maintainability, and deployment simplicity while preserving clear boundaries between presentation, application logic, AI processing, and persistence.

DecisionChoiceRationale
Deployment unitSingle backend monolith + separate SPAOne ASGI process handles REST and WebSocket; frontend built as static assets
Domain organizationFeature modules (auth, exams, session, monitoring, behavior, reports)Each business capability owns models, services, and API layer
Data access patternService–Repository (via Django ORM)Separates business rules from persistence queries
AI integrationAdapter bridge to installable packagebackend/ai/adapter.py → pipeline_playground/knowing_eye
Real-time layerDjango Channels on ASGISame codebase serves HTTP and WebSocket without a separate realtime server

2.1.3 Define Communication Protocols

2.1.3.1 REST APIs for CRUD Operations

Stateless HTTP/JSON endpoints handle authentication, exam management, session lifecycle, monitoring fallback, behavior logs, and reporting. JWT bearer tokens secure authenticated routes. Full endpoint catalog: Chapter 3 and graphs/rest-api-catalog.json.

2.1.3.2 WebSocket for Real-Time Monitoring

Django Channels provides persistent WebSocket connections for frame streaming, analysis feedback, and alert broadcast during live exam sessions. Protocol documented in backend/features/monitoring/consumers.py.

ProtocolUse CaseLatency Requirement
REST (HTTPS)CRUD, auth, submit, reports, frame fallback< 2 s per request
WebSocket (WSS)Live frame analysis, alert push< 300 ms per frame (target)

2.1.4 Design System Deployment Architecture

2.1.4.1 Web-Based Environment

End users access the system through modern browsers on desktop or laptop devices with webcam support. The React SPA is served as static files; API and WebSocket traffic is proxied to the Django ASGI server. No native client installation is required.

2.1.4.2 Cloud Hosting Options Within Scope

Given capstone constraints, deployment targets cost-effective managed or single-VM platforms rather than multi-region infrastructure. See Chapter 6 and graphs/deployment-topology.json.
