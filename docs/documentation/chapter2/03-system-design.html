Chapter II · Part C

System Design

Database structure, data dictionary, and the technologies, models, and algorithms used for data collection through classification in the Knowing Eye platform.

Entity-Relationship Diagram (ERD)

Knowing Eye uses a relational SQL database (SQLite in development, PostgreSQL in production). The ERD reflects Django models under backend/features/.

JSON Schema Diagram

Omitted. Knowing Eye persists structured data in SQL tables via the Django ORM. JSON fields (options, metadata, embedding) are embedded columns validated at the application layer, not standalone JSON document stores. Per capstone guidelines, JSON Schema diagrams are omitted for SQL-based databases.

The API response contract for frame analysis (used by the frontend and WebSocket) follows this structure:

{
  "metrics": { "face_presence_pct", "gaze_focus_pct", "posture_compliance_pct",
               "identity_match_pct", "object_clear_pct", "overall_compliance_pct" },
  "events": [{ "event_type", "score_pct", "confidence_pct", "metadata" }],
  "alerts": [{ "type", "severity", "message", "metric_pct" }]
}

Technologies, Concepts, and Theories

The platform follows a classical computer-vision pipeline—data collection → preprocessing → feature extraction → classification/scoring—embedded in a full-stack web architecture. Technologies are grouped by layer; each entry notes where it is applied in Knowing Eye.

Frontend

React 18 + Vite 6

Single-page application shell. Powers the exam-taking flow (exam-taking-backend.tsx), live monitoring dashboard (monitoring.tsx, session-monitor.tsx), exam builder, and admin reports.

TypeScript

Static typing across frontend/src/. Enforces API contracts in core/config/api.ts and feature modules (auth, exams, monitoring, reports).

React Router 7

Client-side routing with role-gated pages via protected-route.tsx—admin routes (exam builder, monitoring) vs. examinee routes (dashboard, exam session).

TanStack React Query

Server-state caching for reports and session lists (features/reports/hooks/), reducing redundant REST calls on the dashboard.

Tailwind CSS 4 + Radix UI (shadcn/ui)

Design system in shared/components/ui/—forms, dialogs, tables, and charts used across login, exam setup, and monitoring views.

Browser Media APIs

getUserMedia(), Canvas JPEG encoding, and WebSocket in shared/hooks/use-monitoring.ts capture and stream ~1 FPS webcam frames during an exam session.

Backend

Django 6 + DRF

Feature-based modular monolith under backend/features/—authentication, exams, session lifecycle, monitoring REST fallback, behavior logs, and PDF reports.

Django Channels + Daphne (ASGI)

Real-time WebSocket consumers at /ws/monitoring/{session_id}/ receive frames, invoke the AI adapter, and broadcast alerts to admin observers.

PostgreSQL / SQLite

Relational persistence via Django ORM. SQLite in local development; PostgreSQL 15+ in production for users, exams, sessions, behavior logs, and identity embeddings.

Redis + channels-redis

Production channel layer for multi-worker WebSocket broadcast—keeps live alert notifications synchronized when Daphne is scaled horizontally.

ReportLab

Generates downloadable PDF exam and session reports from features/reports/views.py.

AI / Computer Vision

Production pipeline: backend/ai/knowing_eye/, integrated through backend/ai/adapter.py. Pre-trained benchmark accuracies below are from published model evaluations; project-specific UAT metrics are tracked in Part D.

MediaPipe Face & Pose

In project: face_detector.py and pose_detector.py extract 468 face landmarks, head yaw/pitch, and body keypoints for gaze and posture scoring.
Accuracy: Face detection ~98% precision on frontal faces; pose landmark PCK@0.2 ~95% on visible joints (Google benchmarks).

ArcFace / InsightFace (buffalo_l)

In project: arcface_backend.py produces 512-D embeddings; enrolled at exam start and verified each frame via cosine distance (identity_match_threshold: 0.42 in pipeline.yaml).
Accuracy: LFW 99.77%, CFP-FP 98.56%, AgeDB-30 98.18% (InsightFace model zoo).

YOLOv8n (Ultralytics)

In project: Detects persons and prohibited objects (COCO class 67: cell phone) to feed object_clear_pct in the monitoring API schema; configured in pipeline.yaml and assets/graphs/ai-pipeline-component.json.
Accuracy: mAP@50-95 37.3%, mAP@50 52.0% on COCO val (YOLOv8n official benchmark).

OpenCV + NumPy

In project: preprocessing/frame.py handles BGR conversion, resize (≤640 px), denoise, and YCbCr histogram equalization before inference.

Threshold-based behavior classifier

In project: behavior/scoring.py and behavior/temporal.py map detections to 0–100% compliance metrics, rule-based event flags, and sliding-window suspicious-pattern alerts.

Deployment / Server

Nginx

Reverse proxy in production—TLS termination, SPA static hosting (frontend/dist), REST proxy to /api/, and WebSocket upgrade on /ws/.

Daphne / Uvicorn (ASGI)

Serves the Django ASGI application (core/config/asgi.py) on port 8000; managed via systemd in production (docs/deployment.md).

Let's Encrypt TLS

HTTPS/WSS certificates for secure browser camera access (requires secure context) and encrypted API traffic.

Optional CUDA GPU

Accelerates YOLO inference on the server when available; CPU fallback via ONNX Runtime for ArcFace.

Tools / Platforms

Visual Studio Code / Cursor

Primary IDE for Python backend and TypeScript frontend development, debugging, and integrated Git workflows.

Git + GitHub

Version control and collaboration. GitHub Actions validates and deploys the Project OS documentation site on push to main.

Vitest + Django Test Runner

Frontend unit tests (frontend/src/**/*.test.ts); backend API, WebSocket, and AI pipeline tests under backend/features/*/tests/ and backend/ai/tests/.

SheetJS (xlsx)

Bulk question import in the exam builder—admins upload a filled .xlsx worksheet parsed by features/exams/lib/.

Tremor React

Bar, line, and donut charts on the reports page (pages/reports.tsx) for session and violation summaries.

Security

PBKDF2 password hashing

Django's default hasher stores credentials in authentication_user.password—never in plaintext.

JWT (SimpleJWT, HS256)

Access + refresh tokens issued at /api/auth/token/; auto-refreshed by frontend/src/core/config/api.ts. WebSocket auth via query-string token in JWTAuthMiddleware.

RBAC

Role-based access (ADMIN | EXAMINEE) enforced on REST endpoints and UI routes—draft exams and admin dashboards are hidden from examinees.

TLS, HSTS & secure cookies

Production settings (settings/production.py) enable SECURE_SSL_REDIRECT, HSTS, HttpOnly/Secure cookies, and X-Frame-Options: DENY when DEBUG=False.

CORS whitelist

CORS_ALLOWED_ORIGINS restricts API access to the deployed SPA origin; credentials are allowed only from trusted domains.

CV Pipeline Stages

1. Data Collection

Browser getUserMedia() captures webcam video; canvas encodes JPEG/base64 at ~1 FPS. Transport: WebSocket /ws/monitoring/{session_id}/ (REST /api/monitoring/frame/ fallback).

2. Pre-processing

Base64 → BGR; resize to ≤640px; optional denoise; YCbCr histogram equalization. Config: backend/ai/config/pipeline.yaml.

3. Feature Extraction

MediaPipe (468 face landmarks, head yaw/pitch, posture ratios), YOLOv8 object detections, ArcFace identity embedding.

4. Classification & Scoring

Threshold-based 0–100% metrics, rule-based event flags, sliding-window temporal patterns, and weighted aggregation into overall_compliance_pct.
